# Isaac Lab Mimic 機能一覧

## 1. デモンストレーション生成

### 1.1 基本生成（MimicGen アルゴリズム）

MimicGen は人間のデモンストレーションをサブタスク単位に分割し、新しいシーン配置に対して変換・再接合することで合成デモを生成する。

| 手法 | 説明 |
|------|------|
| 線形補間（Linear Interpolation） | サブタスク間の遷移を滑らかに補間 |
| 姿勢変換（Transformation） | エンドエフェクタの軌道を新しいオブジェクト配置に適応 |
| 接合（Stitching） | 変換済みサブタスクを結合して完全な軌道を生成 |
| ランダム選択（Random Selection） | サブタスクセグメントをランダムに選択して組み合わせ |
| 最近傍選択（Nearest Neighbor） | オブジェクト位置またはロボット状態が最も近いセグメントを選択 |

### 1.2 SkillGen（GPU 高速生成）

SkillGen は cuRobo ベースの GPU 並列モーション計画を使った拡張生成手法。

- **テレオペレーション不要**: 自動でデモを生成可能
- **多段階スキル**: アプローチ → コンタクト → リトリートの3フェーズ
- **GPU 並列実行**: cuRobo による高速な衝突回避モーション計画
- **動的オブジェクト管理**: 自動アタッチ/デタッチ + 衝突処理

### 1.3 主要パラメータ

| パラメータ | 説明 | デフォルト |
|-----------|------|-----------|
| `--generation_num_trials` | 生成する合成デモのエピソード数 | 1000 |
| `--num_envs` | 並列シミュレーション環境数 | 5 |
| `--use_skillgen` | SkillGen モードの有効化 | false |
| `--enable_cameras` | カメラデータ付き生成 | false |
| `--headless` | ヘッドレス実行（GUI なし） | false |

### 3.2 カスタムタスクの追加

`ManagerBasedRLMimicEnv` を継承してカスタムタスクを定義可能:

```python
class MyCustomMimicEnv(ManagerBasedRLMimicEnv):
    def get_robot_eef_pose(self):
        """エンドエフェクタの現在姿勢を返す"""
        ...

    def target_eef_pose_to_action(self, target_pose):
        """目標姿勢をアクションに変換"""
        ...

    def get_subtask_term_signals(self):
        """サブタスク終了信号を返す"""
        ...
```

## 4. ポリシー学習

### 4.1 State-based Policy

| 項目 | 詳細 |
|------|------|
| 観測空間 | 関節角度、エンドエフェクタ位置/姿勢、グリッパー状態、オブジェクト位置 |
| アルゴリズム | Behavioral Cloning (BC) |
| ネットワーク | MLP |
| 正規化 | `--normalize_training_actions` 推奨 |

### 4.2 Visuomotor Policy

| 項目 | 詳細 |
|------|------|
| 観測空間 | 状態情報 + カメラ画像（RGB） |
| アルゴリズム | Behavioral Cloning (BC) |
| ネットワーク | CNN エンコーダー（ResNet 系） + MLP |
| カメラ | 単一または複数カメラ対応 |

詳細は [カメラ・画像データの扱い方](./04_camera_and_vision.md) を参照。

### 4.3 学習テクニック

- **Population Based Training (PBT)**: ハイパーパラメータの自動最適化
- **早期終了**: バリデーション性能に基づく早期終了
- **最良モデル選択**: 学習中に最も成功率が高いチェックポイントを自動保存

## 5. Domain Randomization

Domain Randomization（DR）はポリシーの汎化性能を向上させるためにシミュレーション環境のパラメータをランダム化する手法。

### 5.1 物理パラメータ

| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| 摩擦係数 | [0.5, 1.5] | オブジェクトと環境間の摩擦 |
| 質量スケール | [0.8, 1.2] | オブジェクトの質量倍率 |
| 反発係数 | [0.0, 0.3] | 衝突時の反発 |

### 5.2 配置ランダム化

| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| オブジェクト位置 XY | [-0.05, 0.05] m | テーブル上の初期位置のずれ |
| オブジェクト回転 Z | [-π, π] rad | Z 軸周りの回転 |

### 5.3 視覚ランダム化（Visuomotor 用）

| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| 照明強度スケール | [0.5, 2.0] | 光源の明るさ倍率 |
| 照明色温度 | [3000, 8000] K | 暖色〜寒色の変動 |
| オブジェクト色 HSV | 変動あり | 色相・彩度・明度のランダム化 |
| テーブルテクスチャ | ランダム変更 | 背景テクスチャの変更 |
| カメラ FOV | [-2.0, 2.0]° | 視野角の微小変動 |

### 5.4 制御ノイズ

| パラメータ | 範囲 | 説明 |
|-----------|------|------|
| 位置ノイズ | [-0.002, 0.002] m | アクションに加える位置ノイズ |
| 回転ノイズ | [-0.01, 0.01] rad | アクションに加える回転ノイズ |

## 6. 評価機能

- **ロールアウトベースの成功率計算**: N 回のロールアウトのうち成功した割合を計測
- **ヘッドレス評価**: GUI なしでの高速評価
- **カスタムロールアウト数**: `--num_rollouts` で指定（デフォルト: 50）
- **デバイス指定**: `--device cuda` で GPU 上での評価

## 7. データ管理

- **HDF5 フォーマット**: エピソード単位の構造化データ
- **画像データの効率的格納**: チャンク圧縮対応
- **メタデータ**: 環境設定、タスク情報の保存
- **データセット結合**: 複数の HDF5 ファイルをマージ可能

## 8. 実行環境

| 機能 | 説明 |
|------|------|
| ヘッドレス実行 | `--headless` フラグで GUI なし実行 |
| Docker 対応 | 再現性のあるコンテナ環境 |
| Slurm 対応 | ジョブスケジューラによるバッチ実行 |
| GPU 並列 | 複数環境を GPU 上で同時実行 |
| GPU メモリ管理 | State: ~4.5GB/env, Visuomotor: ~8.0GB/env |
